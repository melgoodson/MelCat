Snarky Cat™ Shopify App Tech Buildout


What this app is

This is a simple Shopify app that runs the Snarky Cat digital product system.

It does 5 things:

  1.  gives customers access after they buy 
  2.  handles QR unlocks from physical orders 
  3.  shows them what they own 
  4.  lets them download assets 
  5.  unlocks upgrades and future drops 


That’s it.
The app is not the product.
The app is the engine behind the product.

Main goal:
Use the app to make Snarky Cat work as:

  •  a standalone digital product
  •  a bundle add-on for physical products 
  •  a repeat-purchase / retention tool through future drops 


What the app needs to do:
1) Customer side
The customer should be able to:

  •  buy a pack 
  •  unlock it fast 
  •  see what’s included 
  •  download the files 
  •  upgrade to a higher tier 
  •  get future drops if they qualify 


2) Admin side
You should be able to:

  •  manage packs 
  •  upload assets 
  •  manage QR campaigns 
  •  release drops 
  •  see who owns what 
  •  track basic performance 


The 3 main user flows

1) Standalone purchase flow
Customer buys a digital pack from the store.
Flow:

  •  customer visits product page 
  •  sees previews 
  •  buys Standard / Deluxe / Ultimate 
  •  app gives them access 
  •  customer lands on claim/download page 
  •  app shows their library 
  •  app offers an upgrade 


2) Physical bundle flow
Customer buys a physical product and gets a QR code.
Flow:

  •  customer scans QR code 
  •  app checks the offer 
  •  customer enters email or logs in 
  •  app unlocks free or discounted access 
  •  app offers upgrade to Deluxe or Ultimate 


3) Future drop flow
Customer already owns a qualifying tier.
Flow:

  •  new drop is released 
  •  app checks eligibility 
  •  customer gets access automatically 
  •  email/SMS brings them back 
  •  customer downloads the new assets

  •  upgrade prompt

  •  admin panel for packs/assets/drops

Do not build yet

  •  community features 
  •  gamification 
  •  AI features 
  •  advanced analytics 
  •  anything “cool” that doesn’t increase revenue 

Let’s not build a spaceship to sell cat stickers.

Shopify pieces to use:

1) Theme App Extension
Use this for store pages.

Examples:

  •  preview block 
  •  “what’s inside” section 
  •  animated sample gallery 
  •  upgrade banner 
  •  bundle explainer 


2) App Proxy
Use this so pages feel native to the store.

Examples:

  • /apps/snarky-cat/claim
  • /apps/snarky-cat/library
  • /apps/snarky-cat/upgrade


3) Webhooks
Use these to trigger actions automatically.

Needed for:

  •  order paid 
  •  refund 
  •  customer update 
  •  app uninstall 


4) Metafields / Metaobjects
Use these to organize the content.

Suggested objects:

  •  Pack 
  •  Asset 
  •  Drop 
  •  Tier 
  •  QR Campaign 
  •  Upgrade Offer 


5) Checkout / Post-Purchase Extension
Useful later, not required on day one.

Use for:

  •  post-purchase upsells 
  •  unlock reminder 
  •  order confirmation CTA 


6) Shopify Functions
Optional later.

Use only if you need:

  •  bundle validation 
  •  cart rules 
  •  tier-specific promo logic 


Simple architecture
Frontend
Two parts:
1) Storefront UI
This is what the customer sees on the Shopify site:

  •  previews 
  •  bundle messaging 
  •  upgrade banners 
  •  unlock entry points 


2) App UI
This is where the app runs the main experience:

  •  claim page 
  •  library page 
  •  admin dashboard 


Backend
The backend handles the logic.

It should:

  •  verify purchases 
  •  grant entitlements 
  •  manage tiers 
  •  process QR claims 
  •  control future drops 
  •  track basic analytics 

Keep the backend boring and reliable.
Boring software makes money.

Content delivery rules
The app should clearly show:

  •  what they bought 
  •  what files are included 
  •  how to use them 
  •  what they can unlock next 


Asset types

  •  GIFs 
  •  MP4s 
  •  PNG stickers 
  •  wallpapers 
  •  meme templates 
  •  printables 
  •  install guide PDF 


Tier logic

  •  Lite = small free set 
  •  Standard = main product 
  •  Deluxe = extra content + exclusives 
  •  Ultimate = future drops included 


Rule:
The customer should be able to use it in 5 minutes or less.
If the process is annoying, conversion gets punched in the throat.

QR unlock system
This is one of the best leverage points.

What it does:
It connects physical product sales to digital product adoption.

Flow

  •  QR code goes in the package 
  •  customer scans it 
  •  app identifies the campaign 
  •  app checks the offer 
  •  app unlocks the correct tier 
  •  app tracks the source 


Why it matters
It helps with:

  •  list growth 
  •  post-purchase engagement 
  •  upsells 
  •  repeat visits 
  •  future drop reactivation 

That’s the flywheel.
